# 成绩管理教学平台 —— 三人小组前端开发与答辩工作安排（7天）

## 项目目标
基于 HTML5 + CSS3 + JavaScript 技术栈，完成一个具备完整成绩管理功能的教学平台前端系统，并准备10分钟演示答辩（含PPT）

## 团队构成
- **A（组长）**：负责核心模块、协调进度、主导PPT整合与答辩
- **B、C**：各自独立负责一个功能模块，配合测试与联调

## 🗓 总体时间线（7天）

| 天数 | 主要任务 |
|------|----------|
| 第1天 | 需求拆解、模块分配、技术选型、UI风格统一、建立代码仓库 |
| 第2–4天 | 并行开发各模块（每人独立负责） |
| 第5天 | 模块联调 + 基础数据填充 + 初步集成 |
| 第6天 | 交互优化 + 安全/兼容性检查 + PPT初稿 + 演示脚本编写 |
| 第7天 | 全流程演练 + PPT定稿 + 答辩预演 |

## 🔧 模块划分与责任人

根据作业要求的五大必做模块，结合三人能力，合理分配如下：

| 模块 | 子功能 | 责任人 | 说明 |
|------|--------|--------|------|
| ① 公共前端与用户中心 | - 首页 & 课程浏览（游客/学生）<br>- 账户登录/注册/安全（含忘记密码、错误锁定等） | B | 功能明确、界面标准，适合技术一般成员；使用现成表单验证+本地模拟存储即可 |
| ② 学生端：课程修读与成绩查询 | - 选课/我的课程<br>- 学期成绩总览 + 成绩详情页 | C | 数据展示为主，逻辑较清晰；可复用B的课程列表组件 |
| ③ 教师端 + ④ 教学管理端 + ⑤ 系统管理端（整合） | - 教师：课件上传、作业发布、成绩录入<br>- 教学管理员：班级/课程管理、成绩审核<br>- 系统管理员：日志查看（简化） | A（组长） | 功能复杂、交互多、需状态管理；由技术最强者承担；可适当简化非核心功能（如日志仅展示静态表格） |

✅ 说明：
- 虽然要求5类角色，但前端可共用部分页面结构（如导航栏、布局框架），通过"角色标识"控制显示内容。
- 所有数据采用 本地模拟（JSON + localStorage），不依赖真实后端。
- "教学管理端"和"系统管理端"功能在前端仅做静态展示或简单表单，满足演示即可。

## 🎨 UI/UX 统一规范（第1天确定）

- **设计风格**：高校官网风格 —— 蓝白主色、简洁专业、无花哨动效
- **布局框架**：固定顶部导航 + 左侧菜单（按角色动态切换）+ 主内容区
- **组件库**：手写基础组件（避免引入框架），但统一命名与样式：
  - 按钮：.btn-primary, .btn-outline
  - 表格：.table-striped
  - 表单：统一 label + input 结构
- **响应式**：至少适配 PC（1920×1080）和 iPad（横屏），手机端可降级

## 🛠 各成员每日任务明细

### 第1天：启动日

**全体**：
- 共同阅读需求文档，确认5大角色权限边界
- 确定UI草图（可用Figma/手绘拍照共享）
- 创建GitHub仓库，初始化基础HTML/CSS/JS结构

**A**：
- 设计全局布局（header + sidebar + main）
- 编写 utils.js（含加盐哈希函数、localStorage存取封装）
- 提供模拟账号数据（学生/教师/管理员各1-2个）

**B & C**：
- 熟悉项目结构，拉取代码
- B开始设计登录/注册表单；C开始设计课程卡片组件

### 第2–4天：开发日（并行）

| 成员 | 每日重点任务 |
|------|-------------|
| A | - 实现教师端：课件上传（模拟）、作业发布表单、成绩录入表格<br>- 实现教学管理端：班级/课程管理表格（可编辑）<br>- 实现成绩审核标记逻辑（如优秀率>80%标红）<br>- 编写角色路由逻辑（根据 localStorage.role 切换菜单） |
| B | - 完成首页课程浏览（支持搜索/筛选）<br>- 实现登录/注册/忘记密码流程（含错误锁定模拟）<br>- 实现游客 vs 学生视图差异（如游客不能选课） |
| C | - 实现"我的课程"页面（从模拟数据读取）<br>- 实现学期成绩总览表格 + 点击跳转成绩详情<br>- 复用B的课程卡片，确保样式一致 |

💡 每天下班前15分钟站会同步进度、阻塞问题

### 第5天：集成日

**全体**：
- 合并代码，解决样式冲突
- 填充完整演示数据（至少2个学期、3门课程、5名学生）
- 测试各角色登录后菜单和功能是否正确

**A**：
- 主导集成，修复路由/状态问题
- 添加 loading、错误提示等基础交互反馈

**B & C**：
- 配合测试自己模块在不同角色下的表现

### 第6天：优化 + PPT 初稿

**A**：
- 编写PPT初稿（含：项目概览、分工表、特色功能、演示流程）
- 检查安全性（如密码哈希是否生效）、兼容性（Chrome/Firefox）

**B & C**：
- 优化各自页面的用户体验（如表单回车提交、空状态提示）
- 准备自己负责模块的演示脚本（1分钟内讲清楚）

### 第7天：演练 + 答辩准备

**上午**：
- 全流程演示排练（A操作，B/C补充讲解）
- 录制备用视频（防现场网络/电脑故障）

**下午**：
- PPT定稿（突出：分工明确、功能完整、有细节美感）
- 提交 README（含账号密码示例，如：stu123 / 123456）
- 打包代码 ZIP + PPT，全员备份
